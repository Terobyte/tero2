# MVP2.5 + MVP2.6 Combined Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the full SORA multi-role pipeline (MVP2.5) and TUI dashboard with event bus (MVP2.6) on top of existing MVP0-2 baseline, producing an autonomous agent runner with Scout → Architect → Coach → Builder → Verifier pipeline, plan hardening, context assembly, live TUI visualization, and remote control via Telegram.

**Architecture:** MVP0-2 is fully implemented (~4500 LOC, 23 modules). This plan adds ~40 new files and modifies ~10 existing files. The build order interleaves MVP2.5 and MVP2.6 to avoid rework: shared infrastructure first (state, config, events), then SORA phases, then TUI consumers.

**Tech Stack:** Python 3.11+, asyncio, Textual (optional dep for TUI), requests, pytest, ruff

**Current state of codebase:**
- [x] MVP0: Fully implemented (runner, providers, state, checkpoint, disk_layer, notifier, CLI)
- [x] MVP1: Fully implemented (reflexion, telegram_input, project_init)
- [x] MVP2: Fully implemented (stuck_detection, escalation, runner integration)
- [ ] MVP2.5: Design spec approved, NOT implemented
- [ ] MVP2.6: Design spec approved, NOT implemented

**Key invariants:**
- `DiskLayer` attribute is `sora_dir` (not `sora_path`)
- Lock file is `.sora/runtime/auto.lock` (not `LOCK`)
- `Runner.__init__` currently requires `plan_file: Path` — MVP2.6 makes it optional
- `Runner._build_chain()` is hardcoded to `executor` role — MVP2.5 parameterizes by role
- Escalation state lives on `Runner` instance — MVP2.5 moves it to `RunnerContext`
- `FileLock` already has `__enter__`/`__exit__` — `project_lock.py` adds error translation, not CM interface
- `textual` is optional — all `tero2 run`/`status`/init/telegram` work without it

**Import path convention:** All code uses `tero2.` prefix.

**Known open bugs (15 total in bugs.md):** These are pre-existing. This plan does NOT fix them — they should be addressed in a separate bugfix pass before or after this plan. Notable ones that interact with new code:
- Bug 13: escalation_level cross-plan bleed → partially mitigated by moving state to RunnerContext
- Bug 22: FileLock unlink race → project_lock.py inherits this; fix lock.py first ideally
