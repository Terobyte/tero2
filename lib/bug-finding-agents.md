# Франкенштейн — архитектура дебаггера tero2

> Композитный дебаггер, собранный из лучших техник open-source агентов. Каждый "орган" — одна уникальная техника от одного агента.

---

## Доноры — что взяли (по 1 технике)

| Орган | Техника | Донор |
|---|---|---|
| СКЕЛЕТ | bash loop как основа | mini-swe-agent |
| РУКИ | NL-first CoT: думай на NL → потом код | SRepair |
| АНТИПАМЯТЬ | blacklist неудачных патчей | RepairAgent |
| ИММУНИТЕТ | 3-5 патчей-кандидатов + тесты-арбитр | Agentless |
| PROOF | RED test перед объявлением бага | Strix (принцип) |
| АДАПТАЦИЯ | агент модифицирует свой scaffolding | Live-SWE-agent |
| ФАЗЗИНГ | автогенерация fuzz targets | OSS-Fuzz-Gen |
| Lint-on-Edit | валидация ДО исполнения (-40% ошибок) | SWE-agent |

## Сборка — как органы работают вместе

```
СКЕЛЕТ      — bash loop (prompt → bash → output → repeat)
РУКИ        — NL-first CoT фикс: думай → делай
АНТИПАМЯТЬ  — blacklist неудачных решений
ИММУНИТЕТ   — мультипатч + тест-арбитр
PROOF       — RED test перед объявлением бага
АДАПТАЦИЯ   — self-evolving scaffolding, HARD only
ФАЗЗИНГ    — auto fuzz targets, опционально
♥ СЕРДЦЕ    — автофикс цикл: prove → fix → verify → repeat
```

### ♥ СЕРДЦЕ — автофикс цикл

Клей между всеми органами:

```
RED test (PROOF)
  → CoT: "что сломано и почему" на NL (РУКИ)
  → 3 патча-кандидата (ИММУНИТЕТ)
  → антипамять фильтрует уже пробованные (АНТИПАМЯТЬ)
  → тесты прогоняются на каждом кандидате
  → GREEN = зафикшено
  → всё RED = следующая итерация bash loop (СКЕЛЕТ)
```

**Принцип**: RED test ПЕРВЫЙ. Сначала ДОКАЗЫВАЕМ баг, потом фиксим. Нет RED test — нет бага — не трать ресурсы.

## Effort Levels — 3 уровня глубины

### EASY — секунды
**Когда**: быстрая проверка перед коммитом, CI pre-check
**Органы**: СКЕЛЕТ + РУКИ + PROOF

- Одна итерация, один патч, RED test proof
- Базовые паттерны: null-pointer, off-by-one, obvious injection

### MEDIUM — минуты
**Когда**: code review, pull request, подозрительный модуль
**Органы**: СКЕЛЕТ + РУКИ + АНТИПАМЯТЬ + ИММУНИТЕТ + PROOF

- Запускает тесты, 3 варианта патча, тесты выбирают лучший
- Blacklist неудачных попыток, до 5 итераций
- Пишет RED test для найденного бага

### HARD — десятки минут
**Когда**: критический баг, security audit, важный релиз
**Органы**: ВСЕ + АДАПТАЦИЯ + ФАЗЗИНГ

- Агент модифицирует свой scaffolding под задачу
- До 20-50 итераций с антипамятью
- Фаззинг для нативного кода (если применимо)
- Полный тест-сюит + PR с объяснением
