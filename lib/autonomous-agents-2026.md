# Индустриальные паттерны — основа tero2

> Ключевые принципы из индустрии автономных агентов, на которых построен tero2.

---

## File-Based Memory

Вся память — структурированные Markdown-файлы, считываемые при каждом пробуждении. Не зависит от БД и API, переносима, устойчива к сбоям.

**Главное правило**: никогда не держать всю историю в контексте LLM → экспоненциальный рост затрат + галлюцинации.

> Реализация в tero2: `.sora/` disk layer — STATE.md, DECISIONS.md, KNOWLEDGE.md, session logs.

---

## Heartbeat-архитектура

Агент не должен круглосуточно генерировать токены — это уничтожит бюджет.

```
Агент спит → cron будит → batch-проверка событий
  → есть задача: выполнить → записать состояние → уснуть
```

**Zero-Cost Monitoring**: для длительных процессов агент читает только локальные лог-файлы — LLM не вызывается до завершения или ошибки. Средние затраты: **$0.08 за 24-часовой цикл**.

> Реализация в tero2: Runner (launchd/cron) + Telegram polling.

---

## Code Execution vs Tool Calling

| Характеристика | Tool Calling | Code Execution |
|---|---|---|
| Контекст | Все схемы API загружены заранее | Прогрессивное раскрытие |
| Данные | Через модель | Фильтрация локально |
| Control flow | Каждый шаг = LLM-запрос (дорого) | `for/while` в коде (дёшево) |
| Конфиденциальность | Данные на серверы провайдера | PII обрабатывается локально |

> Реализация в tero2: CLI subprocess execution (Builder/Verifier), не tool-calling.

---

## KPI автономных агентов

| Метрика | Что измеряет |
|---|---|
| **Containment Rate** | % задач без эскалации к человеку |
| **Completion Rate** | % успешных многоэтапных процессов |
| **Confusion Triggers** | Количество логических тупиков и перезапусков |
| **Drift Thresholds** | Деградация качества / выход за рамки полномочий |

> Реализация в tero2: MVP5, модуль metrics — Containment Rate + Completion Rate.

## Ключевая цифра

> General-purpose агент без ограничений сжигает **$300-500/день**. Эффективность = строго ограниченная область + чёткие критерии остановки.

> Реализация в tero2: Budget Guards (tier shifting) + task-scoped context windows.
